## 1. 设计定位与平台

产品定位为"智能周末闲时规划助手"，核心目标是帮助用户在碎片化空闲时间内，快速发现并执行低决策成本的本地活动方案。

**平台：iOS 原生应用，基准机型为 iPhone 17（6.1 英寸屏幕）**

设计关键词：温暖、松弛、可探索、轻决策、生活感。

视觉方向采用"奶油米色 + 琥珀金"城市慢生活风格，遵循 iOS Human Interface Guidelines，强调"像朋友给建议"的体验。

## 2. 品牌气质与情绪板

整体气质：

* 城市周末的松弛感
* 雨天与晴天都可执行的轻量计划感
* 轻复古、低饱和、暖调
* 信息密度中等，避免压迫

情绪参考：

* 咖啡馆木质光影
* 书店几何线条
* 胡同与街巷的在地感
* 本地美食与微旅行的生活颗粒度

## 3. 视觉系统

### 3.1 主色与中性色

建议采用以下色板，便于前端变量化：

```css
:root {
  --brand-primary: #8a7400;
  --brand-primary-strong: #6f5d00;
  --brand-primary-soft: #f1df88;

  --bg-base: #f5f5f7;
  --bg-card: #f4f3ee;
  --bg-chip: #eee5c8;

  --text-primary: #222222;
  --text-secondary: #4a4a4a;
  --text-tertiary: #7d7d7d;

  --line-soft: #d8d2bf;
  --line-strong: #2e2e2e;

  --state-favorite: #cc2b2b;
  --state-success: #4d8a42;
}
```

配色原则：

* 主品牌色只用于关键动作、选中态、核心图标
* 大面积背景保持低对比中性色，避免视觉疲劳
* 功能标签采用柔和浅色底，不抢占主内容注意力

### 3.2 字体与字重

iOS 原生字体推荐：

中文推荐：

* 主字体：PingFang SC（iOS 内置）
* 备选字体：Helvetica Neue

英文与数字推荐：

* 主字体：SF Pro Display（iOS 系统字体）
* 备选字体：Helvetica Neue

字号层级建议：

* 页面主标题：40-44px，SemiBold
* 模块标题：24-28px，SemiBold
* 卡片标题：18-22px，Medium
* 正文描述：14-16px，Regular
* 标签与辅助信息：12-14px，Regular

### 3.3 圆角与描边

圆角体系：

* 页面级容器：24-32px
* 信息卡片：20-24px
* 标签 Chip：12-16px
* 浮动按钮与头像挂件：圆形

描边原则：

* 普通卡片使用 1px 浅色描边
* 需要时间轴和层级强调时使用深色细线
* 禁止高饱和描边与高反差阴影并用

### 3.4 阴影与质感

阴影保持轻量，强调"纸卡叠放"关系：

* 卡片默认：`0 2px 6px rgba(0, 0, 0, 0.06)`
* 悬停或按压反馈：`0 4px 12px rgba(0, 0, 0, 0.1)`
* 底部导航与浮动按钮可用更轻的环境阴影，避免漂浮过高

## 4. 布局规则

### 4.1 栅格与边距

**设计基准：iPhone 17 设计稿 390pt（2x 逻辑分辨率）**

iOS 安全区与边距规则：

* 页面左右边距（横向安全区）：16pt
* 状态栏高度：59pt（含 Face ID 安全区）
* 底部导航高度：83pt（含 Home Indicator 安全区约 34pt）
* 模块间垂直间距：12pt / 16pt / 20pt 三档
* 双列卡片间距：12pt
* 所有内容需遵循 UIEdgeInsets safe area 规范

### 4.2 内容编排

全局页面推荐采用"轻导航 + 场景导语 + 功能分组 + 内容流"的通用结构：

1. 顶部轻导航（菜单、搜索、通知等）
2. 场景化导语卡（如天气、时间段、预算状态）
3. 快速筛选标签组
4. 内容列表区（双列瀑布流或单列时间序）
5. 浮动快捷入口（智能推荐、快速规划、发布）
6. 底部主导航

页面模块可按业务目标自由组合，推荐遵循以下优先级：

1. 用户当前决策最需要的信息
2. 可立即执行的动作入口
3. 历史记录与个性化偏好

## 5. 组件风格规范

### 5.1 顶部导航

* 图标线性风格统一，笔画中等偏细
* 搜索框为"高圆角胶囊"，默认置灰背景
* 顶栏保持留白，不放置过多文字

### 5.2 智能建议卡

* 作为页面第一视觉焦点
* 左侧使用天气或场景图标，右侧放"查看详情"操作
* 文案采用两层结构：主题标题 + 1 行场景建议
* 卡片高度控制在 120-150px，确保首屏容纳更多内容

### 5.3 筛选标签 Chip

* 默认态为浅米色底，深色文字
* 选中态为品牌浅金底，文字加深
* 标签文本不超过 6 个汉字，避免折行

### 5.4 内容卡片

卡片结构建议固定为：

1. 封面图（3:4 或 4:3）
2. 标题（最多 2 行）
3. 副标题或场景描述（最多 2 行）
4. 标签组（1-3 个）
5. 价格或距离信息
6. 收藏与喜欢动作区

视觉规则：

* 卡片内部层级明确，图片与文案区分块
* 信息行距略大于常规，提升呼吸感
* 收藏与喜欢图标保持最小 24px 点击热区

### 5.5 用户信息与偏好模块

* 用户信息区可使用暖黄色底，强化"个人空间"识别
* 偏好项使用图标 + 两行文案表达，避免单行词语过于抽象
* 编辑入口保持轻操作样式，避免主按钮化

### 5.6 记录与时间序卡片

* 涉及历史记录时，可使用左侧纵向时间线增强连续性
* 时间节点使用低饱和灰圆点，避免过强视觉干扰
* 每条记录建议采用"时间 + 内容摘要"组合
* 内容信息可按场景组织为地点、预算、距离、交通方式等字段

### 5.7 底部导航

* 四栏结构：发现、规划、行程、我的
* 选中项用品牌浅金底托底，图标与文字同色强调
* 未选中项保持中性深灰，避免过浅导致可读性下降

## 6. 动效与交互反馈

动效风格应轻柔、短时、可预期，避免眩晕和突兀位移。

建议参数：

* 卡片进入：160-220ms，`ease-out`
* 标签切换：120-160ms，透明度 + 轻缩放
* 底部导航切换：180ms，图标和底托同步过渡
* 收藏与喜欢：100-140ms，微缩放 + 颜色过渡

交互原则（iOS 标准）：

* 重要点击区域最小 44x44pt（iOS Human Interface Guidelines 标准）
* 所有可点击元素必须支持 UIControl 的 highlighted 状态（按压反馈）
* 支持 3D Touch / Haptic Feedback（轻按振动反馈，可选）
* 网络加载使用 UIActivityIndicatorView 或骨架屏，优先还原卡片轮廓
* 导航遵循 iOS 返回手势（右滑返回）与标准导航栏交互

## 7. 内容与文案风格

文案语气应像"本地熟人推荐"，不使用生硬平台话术。

推荐文案原则：

* 标题具象，避免空泛形容词堆叠
* 副标题强调可执行性，如"2 小时可完成""人均 50 元"
* 标签优先场景词，如"雨天室内""City Walk""约会友好"
* 价格、距离、交通方式统一格式，便于快速扫读

## 8. 可访问性要求

* 正文与背景对比度建议不低于 4.5:1
* 重要信息不只依赖颜色区分，需配合图标或文字
* 动效支持系统"减少动态效果"设置
* 图片卡片需保留替代文本与加载失败占位

## 9. iOS 应用开发检查清单

开发联调时可按以下清单验收：

* 所有 UI 元素是否遵循安全区（safeAreaInsets）规范
* 状态栏、导航栏、Tab Bar 的高度与布局是否正确
* 颜色、字号、圆角、间距是否与设计 token 一致
* 卡片是否保持统一层级与信息顺序
* 标签选中态与未选中态是否清晰可辨
* 所有交互是否支持标准 iOS 手势（上拉、下滑、侧滑返回）
* 点击热区是否达到 44x44pt 最小标准
* 按压反馈是否提供（highlighted 状态或 Haptic）
* 时间序或分组结构是否具备稳定的视觉节奏
* VoiceOver 无障碍标签是否完整

## 10. iOS 开发技术建议

**技术栈推荐：**

* SwiftUI（推荐）或 UIKit
* 屏幕分辨率适配：使用 pt 单位而非 px
* 深色模式适配（Dark Mode Support）
* 网络请求使用 URLSession 或 Alamofire
* 本地存储使用 UserDefaults、CoreData 或 FileManager

## 11. MVP 版本建议

MVP 建议先实现以下关键能力：

1. 底部 Tab 导航与页面容器（UITabBarController 或 NavigationStack）
2. 首页：场景导语卡、筛选标签、内容瀑布流
3. 个人页：用户信息、偏好卡、历史记录
4. 收藏、喜欢、分享的基础交互
5. 网络请求与本地数据持久化

后续可扩展：

1. 天气接口联动与场景智能推荐
2. 地图集成与路线规划
3. 推送通知与提醒功能
4. AR 场景预览（可选高级功能）
5. 个性化偏好学习与 ML 推荐
